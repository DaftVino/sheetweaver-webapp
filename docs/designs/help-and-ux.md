## Accepted Scope

### 1. Help Dropdown in Header
Location: `.app-header`, between theme switcher and Copy Debug button.
New element: `<div class="help-menu">` with a `<button>Help ▾</button>` toggle.
Dropdown items (all `target="_blank" rel="noopener"`), grouped with a visual divider:
- "Troubleshooting Guide" → github.com/DaftVino/SheetWeaver-Webapp/blob/main/docs/troubleshooting.md
- "Setup Guide" → github.com/DaftVino/SheetWeaver-Webapp/blob/main/docs/setup-guide.md
- "First-Time User Guide" → github.com/DaftVino/SheetWeaver-Webapp/blob/main/docs/first-time-user-flow.md
- "Changelog" → github.com/DaftVino/SheetWeaver-Webapp/releases
- `<hr role="separator">` divider (1px solid var(--border), margin: var(--space-1) 0)
- "Report a Bug" → github.com/DaftVino/SheetWeaver-Webapp/issues/new/choose
Visual grouping rationale: the first 4 are reference docs; "Report a Bug" is an action. The divider signals the distinction without adding labels.
Closes on outside click (document click listener), closes on Escape key.
IMPORTANT: The Help button click handler must call `event.stopPropagation()` before toggling the
  dropdown state. Without this, the document click handler fires on the same event and immediately
  re-opens the dropdown after the button closes it (classic outside-click loop bug).
Mobile: dropdown must not overflow viewport; position as `right: 0` on the container.
  Min-width: 220px (accommodates longest item text). On viewports narrower than 260px, clamp to
  calc(100vw - var(--space-4)).
CSS class `.landing-only` consideration: Help should be visible on ALL steps, not just landing.
  Do NOT add `landing-only` to any help-menu element.
CSS tokens for `.help-dropdown`:
  background: var(--bg-card); border: 1px solid var(--border); border-radius: var(--radius-md);
  box-shadow: var(--shadow-md); font-size: var(--font-sm);
  Item padding: var(--space-2) var(--space-3); min-height: 44px (touch target).
  Item hover: background: var(--bg-subtle-hover).
  Note: audit body.theme-nes .help-dropdown during implementation — NES theme has special
  .app-header rules (lines 218-223 of Index.html) that may affect dropdown appearance.
Keyboard navigation:
  - Button opens dropdown and moves focus to the FIRST menu item (focus management required).
  - Arrow Down: move to next item; wraps from last to first.
  - Arrow Up: move to previous item; wraps from first to last.
  - Home: jump to first item. End: jump to last item.
  - Escape: close dropdown and return focus to the Help button.
  - Enter on an item: follow the link (default anchor behavior).
  - Space on an item: call `event.preventDefault(); item.click()` — Space does NOT follow `<a href>` links natively (it scrolls the page). Explicit handler required.
  - Tab while menu is open: close the dropdown and let focus proceed naturally. Add `event.preventDefault()` and close logic in the keydown handler. The ARIA `role="menu"` pattern prohibits Tab cycling through items — only Arrow keys move within the menu.

### 2. Copy Debug → Auto-Dismiss Toast
Trigger: successful clipboard write (or _showFallback fallback).
Component: CSS-class-based overlay (NOT `_showFallback()` inline-style pattern). Use a static
  `<div id="debugCopiedToast">` in the HTML (like `#statusToast`), toggled via CSS classes.
  Do NOT dynamically create/destroy DOM — this is a class-toggle pattern, not a createElement pattern.
Position: bottom-center (mirrors `#statusToast` layout). z-index: 10500 (sits between
  statusToast at 11000 and _showFallback at 9999 — all three can coexist without overlap).
CSS structure (extend #statusToast or define a parallel rule):
  position: fixed; bottom: max(var(--space-6), env(safe-area-inset-bottom, 0px));
  left: 50%; transform: translateX(-50%) translateY(10px);
  min-width: 280px; max-width: min(480px, calc(100vw - var(--space-8)));
  background: var(--bg-card); border: 1px solid var(--border); border-radius: var(--radius-md);
  box-shadow: var(--shadow-lg); padding: var(--space-3) var(--space-4);
  opacity: 0; transition: opacity 220ms ease, transform 220ms ease;
  .visible class: opacity: 1; transform: translateX(-50%) translateY(0); pointer-events: auto.
Content:
  - Header row: "✓ Debug info copied!" (left) + close button (right)
  - Body: numbered 3-step guide
    1. Open a GitHub Issue → [Report on GitHub] link (opens issues/new/choose)
    2. Paste the debug text you just copied
    3. Describe what you were doing when the error occurred
  - Optional: "Or check the Troubleshooting Guide first →" link (below steps, smaller font)
Close button: `<button type="button" aria-label="Dismiss notification">×</button>`
  (aria-label required — screen readers announce "times" without it).
Auto-dismiss: 8 seconds (setTimeout + remove .visible class).
Double-click behavior: if Copy Debug is clicked while toast is already visible, REMOVE the
  existing timer, remove .visible immediately, re-add .visible with a fresh 8-second timer.
  Never stack toasts.
Edge case (clipboard unavailable — `_showFallback()` is showing):
  Do NOT show `#debugCopiedToast`. Instead, append instruction content inside `#debugFallbackArea`
  in this DOM order: (existing) note → (existing) textarea → (NEW) numbered 3-step guide → (existing) Close button.
  The instructions appear between the textarea and the Close button so the user reads them
  before dismissing the overlay.

### 3. App Version in Header
Location: near Copy Debug button, subtle styling (muted, non-interactive label).
Value source: **piggyback on the existing loadDashboard() server call** — add `appVersion: APP_VERSION`
to the existing server response. The client-side loadDashboard success handler already runs on page
load; it updates the version label as a side effect. Zero additional RPC, zero quota cost.
Display: `<span id="versionLabel" aria-label="App version">v...</span>` styled as muted text.
Loading state: shows "v..." until loadDashboard() resolves (same timing as connection data).
Note: if the user is on a non-dashboard step (wizard), loadDashboard may not run — version label
can remain "v..." in that case without impact.
Failure state: in the loadDashboard() withFailureHandler, add:
  document.getElementById('versionLabel').textContent = 'v?';
  document.getElementById('versionLabel').setAttribute('aria-label', 'App version: unavailable');
This distinguishes "still loading" (v...) from "RPC failed" (v?). Helps users self-diagnose.

## Edge Cases to Address
- Help dropdown: close on outside click, close on Escape, mobile viewport overflow
- Debug toast: clipboard unavailable path (append to fallback textarea instead)
- Debug toast: rapid double-click on Copy Debug (replace existing toast, don't stack)
- Version display: server-side value not yet loaded (show blank or "..." until snapshot returns)
- Welcome tip vs what's new: mutual exclusion via localStorage key presence
- First-time tip: do NOT show during wizard steps (step-1+); only on dashboard (step-0)
- New doc links in help dropdown: linked before file exists → create doc first

### 8. localStorage Safety Wrappers
All banner localStorage operations MUST use these helpers to handle SecurityError in
Safari private browsing and QuotaExceededError:

  function _lsGet(key) {
    try { return localStorage.getItem(key); } catch(e) { return null; }
  }
  function _lsSet(key, val) {
    try { localStorage.setItem(key, val); } catch(e) { /* silently ignore */ }
  }

If _lsGet returns null (either key absent or storage blocked), treat as "first visit" for
the welcome tip and "no stored version" for What's New. The banner will show on every load
in blocked environments — acceptable UX cost.

### 9. Accessibility Specifications (required in implementation)
Help dropdown:
  - Toggle button: aria-expanded="false/true", aria-haspopup="menu"
  - Dropdown container: role="menu"
  - Each link: role="menuitem"
  - Focus management: on open, call `.focus()` on the first menu item. On Escape close,
    return focus to the Help button (store a reference before opening).
  - Arrow key navigation: Down/Up move between items (wrapping). Home/End go to first/last.
    Implement via keydown listener on the dropdown container, intercepting arrow keys.
  - Touch targets: each item min-height: 44px (already in CSS spec from section 1).
Debug toast:
  - Container: role="status", aria-live="polite" (matches existing #statusToast pattern)
  - Close button: aria-label="Dismiss notification" (required — × symbol has no accessible name)
Version label:
  - span: aria-label="App version: v2.0.0" (updated dynamically when loadDashboard resolves)
  - Loading state aria-label: "App version: loading" (not "App version: v..." which reads confusingly)
Banners (welcome tip):
  - role="note" (NOT role="banner" — banner is a page-level landmark role; using it on an in-step tip creates a spurious landmark in screen reader navigation). Dismiss button is a real `<button>` element.
